● Here's a comprehensive breakdown of the backend:

  ---
  Overall Architecture

  This is a Next.js 16 app using the App Router, deployed as serverless API 
  routes (each app/api/*/route.js is a separate serverless function). It uses
  NextAuth.js v4 for authentication and Prisma ORM with a MySQL database.

  ---
  Data Storage

  MySQL database (defined in prisma/schema.prisma) stores all structured data:

  ┌───────────────────────┬─────────────────────────────────────────────────┐
  │         Model         │                     Purpose                     │
  ├───────────────────────┼─────────────────────────────────────────────────┤
  │ User                  │ User accounts (email, username, hashed          │
  │                       │ password, role)                                 │
  ├───────────────────────┼─────────────────────────────────────────────────┤
  │ PasswordReset         │ OTP-based password reset tokens                 │
  ├───────────────────────┼─────────────────────────────────────────────────┤
  │ Document              │ Metadata for uploaded files (name, URL, type,   │
  │                       │ size)                                           │
  ├───────────────────────┼─────────────────────────────────────────────────┤
  │ Summary               │ Generated summaries (output text, model used,   │
  │                       │ target audience)                                │
  ├───────────────────────┼─────────────────────────────────────────────────┤
  │ SummaryHighlight      │ User-highlighted quotes from summaries          │
  ├───────────────────────┼─────────────────────────────────────────────────┤
  │ ChatThread /          │ Persistent Q&A threads per summary              │
  │ ChatMessage           │                                                 │
  ├───────────────────────┼─────────────────────────────────────────────────┤
  │ QuizSet /             │ AI-generated quizzes linked to summaries        │
  │ QuizQuestion          │                                                 │
  ├───────────────────────┼─────────────────────────────────────────────────┤
  │ SummaryDocument       │ Many-to-many join table linking summaries to    │
  │                       │ source documents                                │
  └───────────────────────┴─────────────────────────────────────────────────┘

  Uploaded files are stored in Vercel Blob (not the local filesystem). The
  Document model stores the Vercel Blob URL in the url field.

  ---
  Authentication (app/api/auth/[...nextauth]/route.js)

  Two sign-in methods via NextAuth.js:
  1. Credentials — email + password, validated with bcryptjs against
  passwordHash in MySQL
  2. Google OAuth — auto-creates a User record on first sign-in (with
  passwordHash: "google-oauth" as a marker)

  The session callback attaches user.id, username, and role from the database
  into the session.

  ---
  Document Upload & Processing Flow

  1. User uploads a file → POST /api/upload sends it to Vercel Blob storage
  2. A Document record is created in MySQL with the Blob URL
  3. POST /api/summarize fetches the file content (PDF via pdf-parse/unpdf, DOCX
   via mammoth, PPTX via pptx2json, XLSX via xlsx)
  4. The extracted text is sent to an LLM (Google Gemini via
  @google/generative-ai, or OpenAI via openai)
  5. The LLM output is saved as a Summary record, with SummaryDocument linking
  it to the source documents

  ---
  In short: MySQL (via Prisma) is the single source of truth for all user data,
  documents metadata, summaries, highlights, chats, and quizzes. Files
  themselves live on Vercel Blob. AI processing happens server-side by calling
  external LLM APIs (Google Gemini / OpenAI).
